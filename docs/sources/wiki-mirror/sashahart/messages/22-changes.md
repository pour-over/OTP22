## About Code 22

Agent code 22 plays a warning message that has slowly evolved over time. It began as a simple warning and grew progressively more elaborate, adding repetitions, emotional weight, and philosophical asides. The evolution has been tracked in detail by the community.

Related codes:
- `222` — "one" and other peculiarities (35 calls)
- `2222` — "Please. Do not dial this agent code. Use agent code 22. You have been warned. Goodbye." (58 calls)
- `22222` — Escalated version
- `2222222` — Honeypot / reduce burden on MLPL instructions (14 calls)

## Later Versions (Post-2013)

The message continued to evolve through 2014-2015, incorporating:
- References to "beautiful"
- Philosophical asides about understanding
- Quotes or fragments from the "Forever Beautiful" lyric motif
- The phrase "How beautiful we are" (also in agent codes 3, 5, 230, 231)

Final documented version (2015-03-24) includes all previous elements plus additional elaboration.

---

## Analysis

The gradual evolution of code 22 is one of the most artistically interesting elements of OTP22. The message:

1. **Starts simple** — a functional warning
2. **Adds history** — "you have been previously warned"
3. **Escalates emotionally** — "ever ever ever"
4. **Adds philosophy** — "You do not understand"
5. **Reaches existential weight** — by late 2013 it's become almost a meditation on futility

The community interpreted this as J (or the system) becoming increasingly frustrated with researchers probing the system — but also potentially as a slow-burn narrative element showing organizational stress.

The visualization at http://303.sashahart.net/messages/22/changes shows a diff-style view of every change.

## Code 222 and 2222

**222** (35 calls): Plays unusual sounds including "one" repeated, electronic tones. Not a straightforward warning.

**2222** (58 calls):
> "Please. Do not dial this agent code. Use agent code 22 for additional warnings. You have been warned. Goodbye."

**2222222** (14 calls):
Contains instructions about establishing a honeypot to reduce burden on MLPL. Possibly instructs agents to create fake versions of the phone system to misdirect researchers.
